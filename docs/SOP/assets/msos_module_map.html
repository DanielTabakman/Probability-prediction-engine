MSOS pillars

Legibility Honest readable market structure — implied distributions, projection, trust labels.
Workflow pillar Edge pillar Legibility pillar T0 contract → T4 workflow hook

Data → processing → output

Raw market and operator inputs flow through Python (fetch, math, serialize). MSOS reads the results — it never recomputes distributions in the browser. Each box below is what it is (label) plus what it means in plain language.

Inputs (what we read)

Deribit options chain BTC, ETH, tier-1 crypto Live calls and puts across strikes and expiries — the starting point for “what price levels the options market is actually pricing in.”
Deribit futures / perp Consistency, horizon forward Forward levels and funding context — anchors time horizons and checks that option-implied forwards agree with listed futures.
Polymarket Cross-venue event probs Crowd-priced yes/no event contracts — compared against options-implied odds in research scans, not live trader execution.
Asset registry / catalog Enabled assets, venues Which symbols are turned on, which venue feeds them, and trust labels — modules only run on assets we have explicitly enabled and witnessed.
MSOS workflow store Theses, paper trades What the trader wrote down — thesis text, saved views, simulated legs — the human intent layer on top of market data.
→

Processing (what Python does)

src/data/* fetchers Pulls fresh quotes from venues, normalizes symbols, and caches with honest as-of timestamps so the UI can label staleness.
src/engine/* math Extracts risk-neutral densities, solves payoffs and strikes, runs consistency checks — all quant work lives here, not in TypeScript.
src/viz/* boundaries Streamlit lab panels and display-API serializers — turns engine objects into stable JSON shapes MSOS can render read-only.
scripts/collect_* VM scheduled archives Scheduled snapshot jobs on the loop host — archives surfaces and status for replay, backtests, and operator evidence; not the live trader path.
→

Outputs (what humans see)

/ppe-display-api/*.json MSOS fetch only Pre-computed, contract-shaped payloads the Next.js app fetches — charts and summary cards without redoing distribution math in the browser.
artifacts/* Ops reports, snapshots Cross-venue scans, surface archives, relay closeout evidence — for operators and research, not primary trader navigation.
apps/msos-web routes Read-only UI Pages traders open — Strategy Lab, Options Horizon, workflow — display and simulation only; no order routing or brokerage hooks.
