Integration tiers — gather info, then take action

Each tier is a ladder: first you can inspect honest output, then you can act on it. Modules stop at the tier they are chartered for — not every tool needs T4.

Tier What you gather (information) What you can do (action) Charter question
T0 Output shape, semantic copy, fixture JSON — “what would this module return?” Approve or reject the contract before UI work; catch wrong labels early What is the output shape?
T1 Live JSON from display boundary — trust fields, as-of, data sources Wire MSOS fetch; verify API matches Streamlit lab; no trader-facing page yet Can MSOS fetch honest JSON?
T2 Visual read in MSOS — charts, summary cards, trust strip Trader acts: relate belief to market, read disagreement, switch asset/expiry, start thesis Can a human inspect and decide in MSOS?
T3 Historical snapshots in artifacts/ — time series for replay and research Operator/research acts: replay past surfaces, run backtests, witness drift; not daily trader nav Why collect? What subset? How often?
T4 Saved workflow hooks — monitor feed, deep-links, command-center cards Trader returns: monitor paper trade, review history, adjust expression after market moves Does it close the trader loop?
T3 rule: No collector slice without a written archive charter (hypothesis, consumer, granularity, retention, non-goals). Do not archive every market tick.

Module cards — gather, act, bottleneck

Every module should answer: what do I learn, what can I do with it, and what is stuck waiting on a human decision.

5 · Cross-venue scan

EVENT_GAP · T3 ops · RESEARCH lane · side channel

Not just data collection. Three automated steps on the VM: (1) snapshot CSV — Polymarket vs Deribit implied probs for matched events; (2) scan — rank gaps and write latest.md + JSON; (3) backtest — score persistence once ≥14 daily snapshots exist. No trader nav page — you read artifacts when researching.

GatherRanked gap report: which event markets disagree most with options-implied odds, with as-of and source notes.
ActResearch only: open artifacts/cross_venue_reports/latest.md, decide if a gap warrants deeper study or a future product slice — not a daily trading workflow today.
BottleneckHistory still accumulating on VM; backtest stays thin until daily task has run ~2 weeks. Optional glance — never blocks Strategy Lab.
