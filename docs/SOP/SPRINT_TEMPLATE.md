# SPRINT_TEMPLATE

Template for a **feature slice** execution spec at `docs/SOP/SPRINT_00X.md` (paths and `## Sprint ID` heading stay as-is for compatibility; a feature slice was formerly called a **sprint**).

## Sprint ID
SPRINT_XXX

## Title
Short, concrete title.

## Objective
What this feature slice should accomplish.

## Execution mode (optional)
- **Full loop (default for higher-risk slices)**: **SELECTION** → **BUILD** → **CLOSEOUT** (and extra **BUILD** if needed)—see `OPERATING_RULES.md`.
- **Compact** (low-risk presentation / copy / layout only): **SELECTION** → one **BUILD** with **integrated closeout** when **Compact slice mode** in `OPERATING_RULES.md` applies.

## Scope in
What is included.

## Scope out
What is not included.

## Files likely to change
List expected files.

## Current behavior to preserve
List important non-regressions.

## Desired behavior
Describe the target state.

## Acceptance criteria
- [ ] Core change works
- [ ] Existing important behavior still works
- [ ] Relevant tests/checks were run
- [ ] Touched-area cleanup is done
- [ ] Risks are reported clearly

## Test plan
List exact commands/checks to run.

## Cleanup expectations
List cleanup allowed/in scope.

## Escalation triggers
List anything that should be surfaced instead of guessed.

## Definition of done
Done means:
- acceptance criteria met
- tests/checks run
- touched-area cleanup complete
- unresolved risks clearly reported
