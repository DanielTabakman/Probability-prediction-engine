**SSOT for verdict → command:** [`PPE_CANONICAL_OPERATOR_SCRIPTS_V1.md`](PPE_CANONICAL_OPERATOR_SCRIPTS_V1.md) · human double-click: [`OPERATOR_BUTTON_MAP.md`](OPERATOR_BUTTON_MAP.md)

**Doc discovery CLI (charter / ad-hoc topics):**

```bash
python scripts/resolve_sop.py --topic "<phrase>" --json
python scripts/resolve_sop.py --chapter <chapter_id> --json
python scripts/resolve_sop.py --module <module_id> --json
python scripts/resolve_sop.py --role <operator|charter|ide_build|explore|neutral> --json
python scripts/resolve_sop.py --list-topics --json
```

| Flag | Purpose |
|------|---------|
| `--topic` | Match phrase → program/SOP bundle (`TOPIC_ROUTES`) |
| `--chapter` | Relay chapter → program + sprint + selection paths |
| `--module` | Module id → `*_PROGRAM_V1.md` |
| `--role` | Thread role → default `load_always` / `never_load` |
| `--list-topics` | Print topic route catalog |

Machine index: [`CHAPTER_DOC_INDEX.json`](CHAPTER_DOC_INDEX.json) (**load on demand for an actual chapter lookup**) · archive catalog: [`ARCHIVE_INDEX.md`](ARCHIVE_INDEX.md) (generated; **do not load for BUILD**) · regenerate: `generate_chapter_doc_index.cmd --write` · stale check: `--check` (gate)

**Chapter mode (CLOSEOUT_ONLY vs IDE_BUILD):** read `Mode:` line in `artifacts/orchestrator/OPERATOR_STATUS.md` — generated by `scripts/ppe_chapter_mode.py`.

## Load order by role

| Role | Open with | Load (in order) | Never load |
|------|-----------|-----------------|------------|
| **operator** | `what's next?` or `THREAD_ROLE: operator` | `OPERATOR_STATUS.md` → `ACTIVE_PRODUCT_DIRECTION.json`; load `AGENT_CONTINUITY_BRIEF.md` only when the context audit reports it fresh and complete | Full UX charter unless asked; do not implement IDE_BUILD in-thread when burst uses `@ppe-director` |
| **ide_build** | `IDE BUILD thread` + `@IDE_BUILD_STARTER_*.md` | Starter file only | `OPERATOR_STATUS`, burst plan, relay queue |
| **charter** | `Charter thread` + one program doc | Stated program/SELECTION doc only | `OPERATOR_STATUS`, `BURST_PLAN.json`, `@ppe-director` |
| **explore** | `Explore thread` / review only | User @ files only | Relay, commits unless asked |
| **neutral** | User question | Answer scope only | Do not lead with operator status |

### Generic founder and charter phrases

ChatGPT Project instructions are already present. A generic `founder charter`, `charter thread`, or `topic thread` opener does **not** justify loading `PHASE_CHAPTER_BACKLOG.json`, `ACTIVE_PRODUCT_DIRECTION.json`, or generated continuity state. Use one compact role contract, then name one relevant program, charter, or GitHub issue. Load backlog/direction state only for explicit selection, closeout, or operator work.

### Generated continuity safety

`AGENT_CONTINUITY_BRIEF.md` is a generated convenience view, not authority over GitHub `main`. Before treating it as startup context, run `token_audit.cmd --stdout` and require all of the following:

- embedded `HEAD` matches the current checkout;
- active relay, sprint, and plan fields are populated;
- no machine-specific absolute path is required to interpret the brief.

If any check fails, do **not** load the brief first. Use current GitHub canon plus `OPERATOR_STATUS.md` / `ACTIVE_PRODUCT_DIRECTION.json` as applicable, then regenerate the brief through `apply_control_closeout_v1`.

### Operator thread — BUILD load order

When implementing product (rare — prefer `@ppe-build-worker`):

1. [`REPO_LAYER_MAP_V1.md`](REPO_LAYER_MAP_V1.md) or BUILD packet `LAYER_PRESET`
2. Active sprint spec
3. Layer frontier: [`MVP1_FRONTIER.md`](MVP1_FRONTIER.md) or [`MSOS_FRONTIER.md`](MSOS_FRONTIER.md)
4. Generated continuity only when the safety checks above pass and the active packet does not already contain the needed state

## Verdict + Mode -> action

Before any relay command on desktop: `python scripts/ppe_loop_host_guard.py --check` — if `"allowed": false`, use desktop row only.

Before any Codex-first product BUILD, run the build-worker preflight from [`WORKER_LANE_POLICY_V1.md`](WORKER_LANE_POLICY_V1.md):

```bat
verify_codex.cmd
verify_build_worker.cmd
python scripts/ppe_worker_lease.py --assess
```

If that preflight fails, record the Codex blocker as a routing/tooling issue and fall back to Cursor only when the active operator profile or blocker requires it.

| Verdict | `Mode` in status | Desktop | VM |
|---------|------------------|---------|-----|
| `IDE_BUILD` | `IDE_BUILD` | Codex-first when `buildWorker=codex`: `DESKTOP_BUILD.cmd` / build-worker handoff -> starter -> gate -> commit -> mark ready; Cursor is fallback/exception | Wait for desktop |
| `IDE_BUILD` | `CLOSEOUT_ONLY` | Do **not** re-implement — finish marker/closeout | `run_ppe_local.cmd` |
| `RUN_LOCAL` | any | **`DESKTOP_CONTINUE.cmd`** | `run_ppe_local.cmd` / `@ppe-finish-worker` |
| `RUN_AUTO` | any | SSH status only | `run_ppe.cmd` or loop |
| Preflight warnings | — | Resolve dirty tree / branch first; [`RECOVERY_PROTOCOL.md`](RECOVERY_PROTOCOL.md) if mixed-plane | Same |

**Common mistake:** `RUN_LOCAL` on desktop ≠ `run_ppe_local.cmd`.

## Factory policy (no skills layer)

PPE uses **rules + generated status + starters + subagents**, not Cursor Agent Skills, for operator workflows. Do not add a parallel skills hierarchy unless a ritual fails after script/rule fixes ([`BUILD_FACTORY_BOUNDARY_V1.md`](BUILD_FACTORY_BOUNDARY_V1.md)).

Built-in Cursor skills (PR babysit, security review) are optional for generic tasks only.

## Anti-patterns

| Avoid | Use instead |
|-------|-------------|
| Opening UX chat with no role | `Charter thread` + program doc |
| `@ppe-ux-director` for brainstorming | Charter + `@ppe-ux-charter` |
| Steward + operator + BUILD in one thread | Separate threads per role |
| Pasting `OPERATOR_STATUS` into charter threads | One-line pointer to operator thread |
| Loading backlog/direction for a generic charter phrase | One role contract + one relevant program or issue |
| Loading stale/incomplete generated continuity first | Current GitHub canon + status/direction; regenerate brief |
| Re-implementing product when `CLOSEOUT_ONLY` | Finish relay / `DESKTOP_CONTINUE` |
